System Architecture

Finishing Labs ERP follows a classic MVC (Model-View-Controller) architecture pattern with some enhancements.

Technology Stack

  • Backend Framework: Flask (Python)
  • Database: SQLAlchemy ORM with SQLite/PostgreSQL (may use raw SQL for complex queries)
  • Frontend: HTML5, CSS3, HTMX
  • Template Engine: Jinja2

Design Principles

  • Component-Based: Reusable template components (sidebar, topbar)
  • HTMX-First: Minimizes JavaScript, server-driven interactions
  • RESTful Routes: Clean URL structure and HTTP verb usage
  • Single Responsibility: Each controller manages one domain

Setup Instructions

Prerequisites

  • Python 3.8 or higher
  • pip (Python package manager)
  • Virtual environment tool (venv or virtualenv)

Installation

Configuration is managed in config.py. Key settings include:

  • SQLALCHEMY_DATABASE_URI: Database connection string

Database Schema

The system uses SQLAlchemy ORM for database interactions. Models are defined in the models/ directory.

Core Entities

  • Customer: Stores customer information and contact details
  • Order: Represents customer orders with items and status
  • PackingSlip: Generated shipping documentation
  • ProductionItem: Tracks items through production stages

Relationships

  • Customer → Orders (one-to-many)
  • Order → PackingSlips (one-to-many)
  • Order → ProductionItems (one-to-many)

Controllers

Controllers handle HTTP requests and coordinate between models and views.

Controller Structure

Each controller is a Flask Blueprint registered in app.py:

from flask import Blueprint, render_template, request

controller_name = Blueprint('name', __name__)

@controller_name.route('/path')
def index():
    # Handle request
    return render_template('template.html')

Template System

Template Architecture

Templates follow a hierarchical inheritance pattern:

  • base_layout.html: Root template with HTML structure
  • Module index.html: Full page with sidebar/topbar
  • Module content.html: HTMX-targeted content only

Component Pattern

Shared UI components (sidebar, topbar) are extracted into templates/components/ for reusability.

{% extends "base_layout.html" %}

{% block content %}
    {% include "components/sidebar.html" %}
    {% include "components/topbar.html" %}
    
    <main>
        {# Page content #}
    </main>
{% endblock %}

HTMX Integration

HTMX enables dynamic, single-page-app-like interactions without writing JavaScript. It works by intercepting user actions and making AJAX requests, then swapping HTML responses into the page.

How It Works

When a user interacts with an HTMX-enhanced element:

  1. HTMX intercepts the action (click, submit, etc.)
  2. Makes an AJAX request with header HX-Request: true
  3. Server detects HTMX header and returns HTML partial
  4. HTMX swaps the response into specified target element
  5. Browser URL updates (if hx-push-url is set)
  6. No page reload occurs - smooth, seamless transition!

Core HTMX Attributes

  • hx-get="/url": Make GET request to URL on trigger
  • hx-post="/url": Make POST request (for forms)
  • hx-delete="/url": Make DELETE request
  • hx-target="#id": Where to insert the response HTML
  • hx-swap="innerHTML": How to insert: innerHTML, outerHTML, beforeend, etc.
  • hx-push-url="true": Update browser URL and history
  • hx-trigger="click": What triggers the request (click, change, load, etc.)
  • hx-confirm="message": Show confirmation dialog before request

Example: Navigation Link

<a href="/purchase-orders" 
   hx-get="/purchase-orders" 
   hx-target="#content" 
   hx-swap="innerHTML" 
   hx-push-url="true">
    Purchase Orders
</a>

{# When clicked:
   1. HTMX GET requests /purchase-orders with HX-Request header
   2. Controller returns purchase_orders/content.html (partial)
   3. HTMX swaps into <div id="content">
   4. URL changes to /purchase-orders
   5. No page reload, sidebar stays in place
#}

Example: Form Submission

<form hx-post="/customers" 
      hx-target="#content" 
      hx-swap="innerHTML">
    <input type="text" name="company_name" required>
    <button type="submit">Create Customer</button>
</form>

{# When submitted:
   1. HTMX POST to /customers with form data
   2. Controller validates and creates customer
   3. Controller returns customer list HTML
   4. HTMX swaps list into #content
   5. User sees updated list, no page reload
#}

Server-Side HTMX Detection

Controllers check for the HX-Request header to determine what to return:

@blueprint.route('/')
def index():
    if request.headers.get('HX-Request'):
        # HTMX request: return content partial only
        return render_template('module/content.html')
    # Direct URL: return full page with sidebar/topbar
    return render_template('module/index.html')

Extending the System

Adding a New Module/Section

This guide walks through adding a complete new section to the system, following our HTMX + Flask architecture pattern.

Step 1: Create the Controller (Backend)

Create a new file in controllers/ - for example, controllers/vendors.py:

"""
Vendors Controller - Vendor management
"""
from flask import Blueprint, render_template, request

# Create blueprint with a unique name
vendors_bp = Blueprint('vendors', __name__)


@vendors_bp.route('/')
def index():
    """
    List all vendors
    
    HTMX Detection Pattern:
    - Check for HX-Request header
    - Return content-only template for HTMX navigation
    - Return full page for direct URL access
    """
    if request.headers.get('HX-Request'):
        # HTMX request: return just the content
        return render_template('vendors/content.html', page_title='Vendors')
    # Direct access: return full page with sidebar/topbar
    return render_template('vendors/index.html', page_title='Vendors')


@vendors_bp.route('/create')
def create():
    """Display create vendor form"""
    if request.headers.get('HX-Request'):
        return render_template('vendors/create_content.html', page_title='New Vendor')
    return render_template('vendors/create.html', page_title='New Vendor')

Step 2: Register the Blueprint

In controllers/__init__.py, add your new blueprint:

def register_blueprints(app):
    # ... existing blueprints ...
    
    # Vendors blueprint - Vendor management
    from .vendors import vendors_bp
    app.register_blueprint(vendors_bp, url_prefix='/vendors')

Step 3: Create Templates

Create a new directory: templates/vendors/

Full Page Template (vendors/index.html):

{% endblock %}

Content-Only Template (vendors/content.html):

Adding a New Feature - Complete Example

Let's walk through adding a "Create Customer" feature, showing the complete HTMX flow from frontend to backend and back.

1. The Frontend: HTMX Button/Form

In your customers/content.html page, add a button that triggers the feature:

<!-- Button to open "New Customer" form -->
<a href="/customers/create" 
   class="btn-new"
   hx-get="/customers/create" 
   hx-target="#content" 
   hx-swap="innerHTML" 
   hx-push-url="true">
    <i class="fas fa-plus-circle"></i>
    New Customer
</a>

{# HTMX Attributes Explained:
   - hx-get="/customers/create" → Makes AJAX GET request to this URL
   - hx-target="#content" → Put response HTML into div with id="content"
   - hx-swap="innerHTML" → Replace inner HTML of target
   - hx-push-url="true" → Update browser URL bar (makes it bookmarkable)
#}

2. The Controller: Handle GET Request

In controllers/customers.py, add a route to return the form:

@customers_bp.route('/create')
def create():
    """
    Display the create customer form
    
    HTMX will request this URL when button is clicked
    We return just the form HTML (no full page)
    """
    if request.headers.get('HX-Request'):
        # Return form content only
        return render_template('customers/create_content.html', 
                             page_title='New Customer')
    
    # Direct URL access: return full page
    return render_template('customers/create.html', 
                         page_title='New Customer')

4. The Controller: Handle POST Request

Add POST handler to controllers/customers.py:

from flask import redirect, url_for, flash

@customers_bp.route('/', methods=['POST'])
def store():
    """
    Process form submission and create customer
    
    This receives the HTMX POST request with form data
    """
    # Get form data
    company_name = request.form.get('company_name')
    contact_email = request.form.get('contact_email')
    phone = request.form.get('phone')
    
    # Validate data
    if not company_name or not contact_email:
        # Return form with errors
        return render_template('customers/create_content.html',
                             error='Company name and email are required',
                             form_data=request.form)
    
    # Create customer record
    # new_customer = Customer(
    #     company_name=company_name,
    #     contact_email=contact_email,
    #     phone=phone
    # )
    # db.session.add(new_customer)
    # db.session.commit()
    
    # Success! Return to customer list
    # HTMX will swap this response into #content
    return render_template('customers/content.html',
                         page_title='Customers',
                         success='Customer created successfully!')

5. The Response: What User Sees

After form submission, the flow completes:

  1. User fills out form and clicks "Create Customer"
  2. HTMX sends POST request to /customers with form data
  3. Controller validates and saves customer
  4. Controller returns customers/content.html (the list view)
  5. HTMX swaps the list view into #content
  6. User sees customer list with success message
  7. No page reload occurred!

      hx-target="#content">
    <input type="text" name="field" />
    <button type="submit">Submit</button>
</form>

Pattern 3: Delete Button

<button hx-delete="/resource/123" 
        hx-confirm="Are you sure?" 
        hx-target="closest tr" 
        hx-swap="outerHTML swap:1s">
    Delete
</button>

Pattern 4: Inline Edit

<div hx-get="/resource/123/edit" 
     hx-target="this" 
     hx-swap="outerHTML">
    Click to edit
</div>

Best Practices

  • Always create two templates: index.html (full page) and content.html (partial)
  • Check for HX-Request header: Return appropriate template based on request type
  • Use hx-push-url: Keep URLs bookmarkable and browser back button functional
  • Follow naming conventions: Blueprint names match folder names
  • Keep controllers thin: Move complex logic to models or service classes
  • Return HTML from controllers: HTMX expects HTML responses, not JSON
  • Use proper HTTP verbs: GET for reading, POST for creating, DELETE for deleting

RESTful URL Conventions

Follow these patterns for consistent API design:

  • GET /resource - List all items
  • GET /resource/create - Display create form
  • POST /resource - Create new item
  • GET /resource/:id - View single item
  • GET /resource/:id/edit - Display edit form
  • POST /resource/:id - Update item
  • DELETE /resource/:id - Delete item

Troubleshooting Guide

  • Nothing happens when clicking: Check browser console for HTMX errors, verify HTMX script is loaded
  • Full page reload occurs: Missing hx-get or hx-post attribute on element
  • Content doesn't appear: Verify hx-target points to existing element ID
  • URL doesn't update: Add hx-push-url="true" to navigation links
  • Wrong template returned: Check request.headers.get('HX-Request') logic in controller
