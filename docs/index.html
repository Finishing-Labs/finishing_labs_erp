Key Features

  • Customer Management: Customer profiles, contact information, and order history
  • Order Processing: Track From PO to Delivery
  • Packing Slips: Generate packing slips
  • Production Management: Track production status and build orders efficiently

Technology Stack

  • Backend: Python, Flask
  • Frontend: HTML, CSS, HTMX
  • Database: SQLite/PostgreSQL
  • Architecture: MVC pattern with component-based templates

Phase One

Phase One is all about getting the core functionality up and running. We're building complete CRUD operations (Create, Read, Update, Delete) for all the main modules.

What's CRUD?

CRUD stands for Create, Read, Update, Delete — basically the four things you need to do with data:

  • Create: Add new stuff (like a new customer or order)
  • Read: View what you've got (lists, details, etc.)
  • Update: Edit existing records when things change
  • Delete: Remove things you don't need anymore

What's Included?

Phase One covers the essential modules you need to run your finishing lab:

  • Customers: Store all your customer info, contacts, and history
  • Purchase Orders: Create and manage purchase orders
  • Line Items: Manage individual items within each order
  • Work Orders: Create and track production jobs
  • Packing Slips: Generate packing slips

Once Phase One is done, you'll have a system for tracking jobs. Future phases will add more advanced features
